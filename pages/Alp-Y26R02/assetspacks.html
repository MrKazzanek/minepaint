AssetsPacks

The AssetsPack system works similarly to mods and resource packs in Minecraft, but it allows adding packs that can:

  • Change textures of existing blocks, objects, and weather overlays
  • Add custom blocks, blockstates, and new categories
  • Edit existing blocks, blockstates, and categories
  • Rearrange blocks and categories (both custom and existing) in the inventory
  • Remove blocks and categories

However, the system does not allow modifying program functions, adding custom tools, etc., which is why it is not called a mod.

How to Install an AssetsPack?

To install a custom AssetsPack, click the AP button, which opens the AssetsPack manager. You will see a list of available packs and those already enabled. Below the list, there are two buttons: Refresh List to refresh the list and reload pack contents, and Open Folder to open the folder where you should place the AssetsPack folder. After adding a pack, refresh the list using the Refresh List button. You can enable multiple packs at the same time.


Packs can also be added directly through the file explorer to the folder:

C:\Users\User\AppData\Roaming\Godot\app_userdata\MinePaint\assets_packs

How to Enable a Selected AssetsPack?

To activate a chosen pack, open the AssetsPack manager with the AP button and check the checkbox next to the desired pack. When a pack is enabled, its tile will be highlighted in green and the checkbox marked. Up/Down arrow buttons will also appear, allowing you to set the order of enabled packs. A pack higher in the order has priority – for example, if two packs modify the texture of the dirt block, the texture from the pack higher in the list will be applied.
