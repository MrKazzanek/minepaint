Pack Structure

AssetsPacks are essentially just folders with content. Their structure looks like this:

ExampleAssetsPack/
├── assets/
│   ├── blocks/
│   │   └── example_block_texture.png
│   ├── objects/
│   │   ├── moon/
│   │   │   └── moon_0.png
│   │   └── sun/
│   │       └── sun.png
│   └── weather/
│       └── rain.png
├── data/
│   ├── blocks/
│   │   └── example_block.json
│   ├── blocks_order.txt
│   └── categories.json
└── pack.json (required)

The pack.json file

The pack.json file is required for the AssetsPack to be visible in the program.

Worth Knowing

If an AssetsPack is missing a folder or file, it’s not a problem – the program will use its default assets instead.
