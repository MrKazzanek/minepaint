Pack File

The pack.json file acts as instructions for the program – it tells the program that a given folder is an AssetsPack. This file is required; without it, the AssetsPack will not be detected or usable in the program. It allows the program to know the pack’s name, version, and (optionally) a description of its contents.

Keys in the pack.json File

Below are all the keys that can be included in the pack.json file and their meanings:

Key Description
name Defines the name of the AssetsPack. This is a required key because the program uses it to identify the pack.
description An optional key that allows adding a short description of the pack – e.g., information about its contents, author, or purpose.
version An optional key that specifies the version of the pack.

Example pack.json Content

The pack.json file uses the JSON format (JavaScript Object Notation), which is easy to read and edit. Each key and its value is written as a key: value pair. Below is an example: 

{
    "name": "VanillaAssetsPack",
    "description": "Default MinePaint resources as an AssetsPack | Author: QozaWorks",
    "version": "Alp-Y26R02"
}

In this example:

  • name – pack name: VanillaAssetsPack
  • description – pack description: Default MinePaint resources...
  • version – pack version: Alp-Y26R02
This structure allows the program to easily load the pack and display its information in the AssetsPack manager.
